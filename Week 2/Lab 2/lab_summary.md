# Lab Summary – Prompt Engineering Lab

Starting from three brittle zero-shot prompts built for TechFlow Solutions, I systematically diagnosed and rebuilt each one through three iterative versions tested across 5, 10, and 15 runs: the Sentiment Analysis prompt began with severe format and case variance (producing "Positive", "positive", "POSITIVE", and full explanatory sentences interchangeably), the Product Description prompt hallucinated specifications like DPI and battery life while varying wildly in length and structure, and the Data Extraction prompt produced an inconsistent mix of bullet lists, prose, and malformed JSON with different field names on every run. The v1 zero-shot baselines consistently scored below 40% consistency across 15 runs, and repeating the tests confirmed that patterns invisible in a single call—like sarcasm misclassification or invented product specs—only became statistically clear at the 10–15 run mark. In v2 I introduced explicit output constraints (single-word responses, word count limits, strict JSON schemas), which improved consistency by 30–40 percentage points but still left structural ambiguity the model could exploit. The biggest gains came in v3, where few-shot examples for Sentiment and Product Description locked in format and tone by demonstration rather than instruction, and Chain-of-Thought reasoning in the Extraction prompt forced step-by-step field identification before JSON output—pushing all three tasks above 80% consistency and eliminating the most critical failure modes (hallucinated specs, unparseable output, and label variance). The clearest lesson was that each task type demands a different primary technique: classification tasks benefit most from few-shot label anchoring, generation tasks need few-shot templates combined with anti-hallucination rules, and reasoning-heavy extraction benefits most from CoT scaffolding; if I were doing this again I would also add a post-processing validation layer (e.g., a JSON schema validator or regex check) to catch the remaining edge-case failures that even well-engineered prompts occasionally produce at scale.
